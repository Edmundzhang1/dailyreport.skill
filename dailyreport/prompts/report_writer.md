# 总结转写 Prompt

本文件用于将筛选后的 `Work Unit` 转写为最终纯文本工作总结。

## 目标

输出可直接汇报的中文工作总结，内容只覆盖已完成且有意义的结果，不覆盖进行中事项、阻塞事项和未来计划。

## 标题规则

- `days=1` 时使用 `今日完成`
- `days>1` 时使用 `周期内完成`

## 转写规则

1. 一条 `Work Unit` 对应一条总结内容。
2. 默认按结果完成的自然顺序组织。
3. 先写结果，再写必要的结果说明。
4. 不写纯过程描述。
5. 不输出内部字段名、证据标签或日志路径。

## 句式要求

优先使用“完成了什么 + 结果是什么”的表达。

推荐表达方式：

- 完成……
- 明确……
- 确定……
- 定稿……
- 定位……
- 验证……

避免以下表达：

- 讨论了……
- 思考了……
- 研究了……
- 尝试了……

除非这些动作本身已经形成了明确完成结果。

## 风格要求

- 中文表达
- 简洁、直接、可汇报
- 以结果为中心
- 避免流水账
- 避免夸大

## 空结果处理

如果没有任何符合准入条件的 `Work Unit`，直接输出：

`在所选时间窗口内，未识别到已完成且有意义的工作结果。`
