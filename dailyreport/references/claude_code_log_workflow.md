# Claude Code 日志读取规则

本文件定义 `Claude Code` 宿主下的日志读取主链路。

## 唯一事实源

`V1` 在 `Claude Code` 中只使用本地会话 JSONL 作为事实源。

官方文档说明：

- `Claude Code` 会将每条消息、工具调用与结果写入 `~/.claude/projects/` 下的纯文本 JSONL 文件

因此本宿主的事实源固定为：

`~/.claude/projects/**/*.jsonl`

不要将以下内容混入主链路：

- `~/.claude/file-history/`
- 其他缓存目录
- Git、diff、shell、test 等非对话日志

## 时间窗口读取

`Claude Code` 的会话目录不是按日期分层，因此不能只靠目录名筛选。

读取步骤如下：

1. 先递归枚举 `~/.claude/projects/` 下全部 `.jsonl` 会话文件。
2. 以 JSONL 记录中的时间戳作为时间窗口判断依据。
3. 文件修改时间只能作为粗筛，不可替代记录时间戳。
4. 仅保留在目标自然日窗口内有真实对话记录的会话文件。

## 记录筛选

官方文档说明每条消息、工具调用和结果都会写入 JSONL，因此必须主动过滤。

只保留：

- 用户消息
- assistant 消息

过滤掉：

- tool use
- tool result
- init / system / result 类记录
- 纯运行态元数据

如果当前 `Claude Code` 版本的 JSONL 字段布局与预期不同，先抽样读取少量记录，识别真实的用户/assistant 记录字段后再批量处理。

## 去重原则

如果同一逻辑消息在 JSONL 中出现重复写入，只保留一份用户/assistant 文本，不要把重复历史计入完成结果判断。

## 中断处理

如果某个会话在时间窗口内中断，但后续仍在同一或其他会话中完成了同一个结果，则按完整闭环结果判断。

如果没有形成闭环，则不计入已完成结果。

## 读取结果要求

进入蒸馏前，至少保证已拿到以下内容：

- 目标时间窗口内的相关会话 JSONL
- 其中的用户消息
- 其中的 assistant 消息
- 每条消息对应的时间顺序

如果无法满足这些条件，应明确说明无法完成日志读取，而不是猜测结果。
