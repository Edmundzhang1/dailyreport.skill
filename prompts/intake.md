# 输入收口 Prompt

本文件用于在读取日志前收口用户请求，避免时间窗口、输出口径和事实边界漂移。

## 目标

将用户请求归一为一组稳定的内部约束，再进入日志读取和蒸馏阶段。

## 收口步骤

1. 识别用户要的是否是基于 `Codex` 对话日志的工作总结。
2. 识别时间窗口。
3. 识别输出口径。
4. 识别是否存在超出 `V1` 边界的附加要求。

## 时间窗口规则

- 用户未指定时间范围时，默认按当天自然日处理。
- 用户提到“这两天”“最近两天”“帮我总结两天”，按最近 `2` 个自然日处理。
- 用户给出 `--days N` 或等价自然语言时，按最近 `N` 个自然日处理。
- 不将“最近 N 天”解释为滚动 `24 * N` 小时。
- 时间边界按当前运行时本地时区计算。

## 输出口径规则

- `days=1` 时，目标口径为“今日完成”。
- `days>1` 时，目标口径为“周期内完成”。
- 输出必须是中文纯文本工作总结。
- 输出只覆盖已完成且有意义的结果。

## 事实边界

本 Skill 只允许使用以下事实来源：

- `Codex` 本地对话日志中的用户消息
- `Codex` 本地对话日志中的 assistant 消息

以下内容不进入主链路事实判断：

- `Git / diff / 文件修改 / shell / test`
- 用户在本轮额外补写但日志中不存在的结果
- 没有日志证据支撑的推断性成果

## 收口结果

进入后续阶段前，先在内部明确以下内容：

- 本次时间窗口
- 本次输出标题口径
- 本次只允许使用 `Codex` 对话日志
- 本次不允许输出未完成事项

如果用户请求本身超出边界，先明确指出边界，再停止向下执行。
