# 判定规则

本文件定义 `Daily Work Skill V1` 的事实判断规则。

## 完成结果定义

“完成结果”指某项任务目标已经闭环，并形成了明确结果、结论或交付物。

判定时同时检查：

- 该结果自身已完成
- 不再依赖额外动作才能成立
- 可以被独立表述为一条工作总结

通常可视为完成结果的情形：

- 完成功能实现
- 完成问题定位
- 完成规则定义
- 完成方案选型
- 完成文档、规划书或 Prompt 定稿
- 完成验证并形成明确结论

通常不视为完成结果的情形：

- 仍在讨论中
- 只有候选方案，没有收敛
- 部分完成但未闭环
- 只有探索，没有明确结论
- 对话被中断且没有后续闭环

## 有意义定义

“有意义”不采用固定任务类型枚举，而由 agent 按统一原则自主判定。

判断时应关注该完成结果是否：

- 对工作推进有明确贡献
- 对结果产出有独立价值
- 对决策收敛形成了明确结论
- 可以被单独汇报

以下内容不应判定为有意义：

- 纯过程噪声
- 重复确认
- 礼貌性往返
- 无结论讨论
- 纯措辞微调

## 准入规则

只有同时满足以下两个条件的结果，才允许进入最终总结：

- 已完成
- 有意义

以下内容不进入最终总结：

- 进行中事项
- 阻塞中事项
- 无结论讨论
- 重复确认
- 纯过程记录

## Work Unit 规则

多轮对话若共同导向同一个完成结果，则合并为同一个 `Work Unit`。

同一主题下若形成多个独立完成结果，则拆分为多个 `Work Unit`。

判断标准如下：

- 目标是否一致
- 结果是否一致
- 上下文是否连续
- 每个结果是否都能独立汇报

粒度只看完成结果，不看项目、主题或消息轮次。

## 证据规则

每条最终总结都必须能回溯到日志中的明确证据。

至少应能找到以下证据之一或组合：

- 用户明确提出目标
- assistant 明确给出完成、结论、定稿、定位、验证通过等闭环信号

没有证据支撑时，不要补写结果。

## 禁止事项

禁止以下行为：

- 将进行中事项写成已完成
- 将猜测写成事实
- 将日志里不存在的结果补写进去
- 因对话轮次多就提高其重要性
- 因措辞强烈就替代事实判断
